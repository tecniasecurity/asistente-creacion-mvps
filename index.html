FORMATO:

  1. Utiliza la pagina “tecniasecurity.com” como referencia para el layout del prototipo.

    1. El controlador será un servlet de nombre “Controller”, cuyo URI será “/controller/*”, que invoque “Acciones” representadas en el backend por una interfaz “Action” que recibe los datos de cada petición y los procesará definiendo este método:
      public Object run (HttpServletRequest request, HttpServletResponse response) throws ServletException;
    2. Cada “Acción” que implemente esta interfaz, representará un microservicio que el frontend invocará.
    3. El “modelo” serán las clases que implementan la lógica de negocio.
  3. La aplicación implementará un pool de conexiones utilizando la API HikariCP.

●       ROLLBACKCHECKPOINT: Si ocurre un error irreparable, puedo volver a la versión guardada usando “SAVECHECKPOINT”
